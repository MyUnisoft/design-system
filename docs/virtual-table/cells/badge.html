Menu#

One of the main purpose of this typeCell is to display a menu based on the current value.

SwitchBagde#

The SwitchBadge cell allows to handle advanced needs, like 3 dimensionnals datas.

Example#

In this example, we will display the grades of students during the year and for multiple subject.

'No thanksgiving for you' : 'Send mail to parent' }], spring: [{ status: ['ok', 'error'], label: 'Enjoy your spring break' }] } const config = { columns: [ { header: 'Name', key: 'name', size: 1 }, { header: 'Winter', key: 'grades.winter', typeCell: TYPE_CELL.SWITCH_BADGE_STATUS, switchKey, actionsByStatus: actionsStatus.winter }, { header: 'Spring', key: 'grades.spring', typeCell: TYPE_CELL.SWITCH_BADGE_STATUS, switchKey, actionsByStatus: actionsStatus.spring }, { header: 'Summer', key: 'grades.summer', typeCell: TYPE_CELL.SWITCH_BADGE_STATUS, switchKey, actionsByStatus: [] } ] } const handleChange = (event) => { setSwitchKey(event.target.value); }; return ( <div> <Select value={switchKey} onChange={handleChange} > <MenuItem value="history">History</MenuItem> <MenuItem value="math">Math</MenuItem> <MenuItem value="sport">Sport</MenuItem> </Select> <VirtualTable data={data} config={config} tableKeyName="doc-switchBadgeGradeExample" /> </div> ) } export default SwitchBadgeGradeExample;
tip

This could be achieve without using SwithBadge by using a dynamic data object. But the advantage of SwitchBadge is to only update the config props. This can be very useful with a large amount of data.
